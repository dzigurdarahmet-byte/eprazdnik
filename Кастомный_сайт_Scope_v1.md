# Кастомный сайт-каталог «Е-Праздник» — детальный scope v1

Документ для согласования объёма работ с клиентом и понимания как это будет выглядеть.

---

## Что делаем одной строкой

Делаем публичный сайт-каталог программ агентства по образцу HTML-прототипа, который показывали в начале. Контент сайта тянется из Notion-базы — той самой, что мы сейчас собираем по основному контракту. Клиент редактирует программы в Notion → сайт обновляется автоматически.

---

## Архитектура (как это всё связано)

```
┌─────────────────┐        ┌──────────────────┐        ┌──────────────────┐
│   Notion-база   │ ─API─→ │  React-сайт      │ ─────→ │   Пользователь   │
│  (наша работа   │        │  (новый кастом)  │        │  (клиент агентств│
│   по контракту) │        │                  │        │   а или менеджер)│
└─────────────────┘        └──────────────────┘        └──────────────────┘
       ↑
       │ редактирует здесь
   ┌───────────┐
   │  Клиент   │
   │ агентства │
   └───────────┘
```

**Ключевая идея:** клиент НЕ работает в новой системе. Он редактирует программы в Notion как сейчас. Сайт — это просто красивая публичная морда поверх Notion.

## Экран 1. Главная (Dashboard)

**Что видит пользователь:**

- Большой приветственный заголовок («Добро пожаловать в каталог Е-Праздник» или с именем менеджера, как в прототипе)
- 4 цветные карточки быстрых переходов (Quick Actions):
  - 🎯 Каталог услуг (86 программ и шоу)
  - 📂 Мои подборки → ведёт в Notion-страницу с подборками
  - 🔥 Скрипты продаж → ведёт в Notion-страницу со скриптами
  - 💼 Прайсы и КП → ведёт в Notion-страницу с прайсами
- Блок «Последние программы» — 3-4 последние добавленные/обновлённые карточки из Notion
- Поиск программ из любой точки страницы (хоткей `⌘K`)

**Что работает:**

- Клик по карточке Quick Action — переход на нужный экран или внешнюю ссылку
- Клик по карточке программы из «Последние» — переход на детальную страницу
- Поиск работает с глобальной строки (находит по названию, тегам, описанию)

**Что НЕ войдёт:**

- AI-подбор программы по фильтрам типа «детский праздник, 8-12 лет, 50-100к ₽» — это отдельная фича с подключением LLM
- Персональная статистика менеджера («12 встреч на неделе», «5 активных подборок») — это требует авторизации и базы пользователей

## Экран 2. Каталог

**Что видит пользователь:**

- Заголовок «Каталог услуг» + счётчик («86 программ и шоу для любого формата»)
- Поисковая строка с хоткеем `⌘K`
- Чипсы-фильтры по категориям (Шоу, Квесты, Мастер-классы, B2B, B2C, дети, взрослые) — по клику фильтруют выдачу
- Gallery карточек программ — большие карточки с обложкой, эмодзи-иконкой, названием, подзаголовком, бейджами (Готово, Сюжетная, КРИО, премиум, и тд)
- При наведении на карточку — лёгкий hover (тень, чуть приподнимается)

**Что работает:**

- Поиск находит программы по названию и описанию
- Фильтры по чипсам — реальная фильтрация без перезагрузки
- Клик по карточке — переход на детальную страницу
- Когда ничего не нашлось — «Ничего не найдено по запросу X»

**Откуда контент:**

Все карточки рендерятся из Notion-базы «Программы». Когда клиент добавляет новую программу или меняет статус/теги — в каталоге появляется автоматом за 1-2 минуты.

## Экран 3. Страница программы

**Что видит пользователь:**

**Hero (верх):**

- Большая обложка слева (фото или цветной паттерн) с эмодзи-иконкой
- Бейджи поверх обложки: «Готово к продаже», «Сюжетная», «КРИО», и тд
- Кнопки внизу обложки: «Видео-демо 1:24», «48 фото» (модалки)
- Справа от обложки — заголовок программы, подзаголовок
- Цветные теги (готово, сюжетная, КРИО, премиум)
- Метрики в ряд: Возраст / Длительность / Гостей / Стоимость от
- CTA-кнопки: «Отправить клиенту» (открывает форму или mailto), «В подборку» (визуальная кнопка, в первой версии без сохранения state)

**Табы (под Hero):**

5 табов с содержимым программы:

1. **Описание** — легенда программы, развёрнутый текст
2. **Состав программы** — что входит: 7-10 активностей с цветными иконками и описанием. Плюс блок «Персонажи» с аватарами
3. **Расчёт** — таблица цены и условий (база, доп. гость, спецэффекты, локация в МО, предоплата) + кнопка «Открыть калькулятор» (ведёт в Google Sheets)
4. **Медиа** — плитки с медиа-материалами: Reels для Stories, презентация PDF, фото-сет, отзывы клиентов — все с прямыми ссылками на Я.Диск
5. **Скрипты продаж** — текстовое содержание скрипта для этой программы (опционально, если у клиента есть скрипты в Notion)

**Aside (правый блок, фиксированный):**

- Карточка «Цена и условия» — те же цифры что и в табе Расчёт
- Карточка «Маркетинг-кит» — 4-5 файлов для отправки клиенту (Reels, презентация, фото-сет, отзывы)
- Карточка «Менеджер программы» — аватар + имя + кнопка «Написать» (mailto или открытие чата)

**Низ страницы:**

- Блок «Похожие программы» — 3 карточки из той же категории

**Что работает:**

- Переключение между табами без перезагрузки страницы
- Клик по медиа-плитке — открывает Я.Диск в новой вкладке
- Кнопка «Открыть калькулятор» — открывает Google Sheets таблицу из Notion (та самая что мы готовим на Day 3)
- Клик по карточке в «Похожих» — переход на её страницу

**Откуда контент:**

Вся страница рендерится из Notion-карточки конкретной программы. Меняешь в Notion легенду — на сайте обновилось.

---

## Что обновляется автоматически из Notion

Когда клиент редактирует Notion-карточку программы:

- Меняется название, подзаголовок, легенда, активности, персонажи → обновляется на сайте
- Меняется статус «Готово к продаже» → меняется бейдж на карточке
- Меняется цена → обновляется в Hero и в Расчёте
- Добавляется новая программа в Notion → появляется в каталоге
- Архивируется программа в Notion → исчезает с сайта

Время обновления: 1-2 минуты (revalidation через Notion API).

## Что НЕ входит в этот объём

Чтобы избежать недопонимания, явно фиксирую что в 36 000 НЕ войдёт:

- **AI-подбор услуги по фильтрам** (модель типа «детский праздник, 8-12 лет, бюджет 50-100к → топ 3 программы») — требует LLM-интеграции и эмбеддингов, отдельная работа
- **Авторизация менеджеров с собственным кабинетом** — личные подборки, отметки «избранное», статистика встреч
- **Подборки для отправки клиентам через сайт** (сейчас обходимся ссылкой на Notion-страницу с подборками)
- **Кастомная админ-панель для редактирования контента** — она не нужна, редактирование идёт в Notion
- **CRM, заказы, оплата, бронирование** — это уже отдельный полноценный SaaS
- **Мобильное приложение** — только адаптивный сайт

## Технический стек (для понимания)

- **Фронтенд:** Next.js 14 + React 18 + Tailwind CSS — современный, быстрый, SEO-friendly
- **Источник данных:** Notion API (читает базу программ + базу элементов)
- **Хостинг:** Vercel — бесплатный тариф достаточен для каталога такого размера
- **Домен:** клиент покупает отдельно (~1500 ₽/год) и подключает к Vercel — помогу с настройкой
- **Аналитика:** базовая Яндекс.Метрика на всех экранах

## Этапы и сроки

**Старт:** после согласования scope и оплаты второй части (18 000 ₽).

**Параллельно с этим:** Notion-каталог по текущему контракту доделывается как договорились (раскатка структуры на 15 программ + Google Sheets на Day 3).

**Этап 1. Setup (3 дня):**

- Запуск Next.js-проекта, подключение Notion API
- Настройка типов данных, парсинг Notion-блоков
- Базовый роутинг (Главная / Каталог / Программа)

**Этап 2. Каталог + Программа (5-7 дней):**

- Вёрстка экрана Каталога с поиском и фильтрами
- Вёрстка детальной страницы Программы со всеми табами
- Подключение реальных данных из Notion
- Кастомные шрифты, типографика, hover-эффекты — точно как в прототипе

**Этап 3. Главная + полировка (2-3 дня):**

- Вёрстка Главной с Quick Actions
- Блок «Последние программы»
- Глобальный поиск
- Адаптив под мобильные

**Этап 4. Деплой и передача (1-2 дня):**

- Деплой на Vercel
- Подключение домена клиента
- Настройка Яндекс.Метрики
- Демо-показ работы

**Общий срок: 2 недели** с момента согласования. Если параллельно идёт Notion-каталог — может занять чуть больше календарного времени, потому что разработка идёт вечерами.

## Что получает клиент по завершении

1. **Работающий публичный сайт** по адресу выбранного клиентом домена (например, e-prazdnik.ru или katalog.e-prazdnik.ru)
2. **Доступ к проекту в Vercel** — клиент видит метрики, может управлять деплоями
3. **Исходный код проекта на GitHub** — клиент получает приватный репозиторий с полным кодом, может в будущем нанять любого React-разработчика для доработок
4. **Документ-инструкция** — как редактировать программы в Notion чтобы это отражалось на сайте, типичные сценарии (добавил программу, поменял цену, обновил медиа)
5. **Демо-сессия по Zoom** — показываю как всё устроено, отвечаю на вопросы (1-1.5 часа)
6. **30 дней поддержки после сдачи** — если что-то сломалось не по вине клиента, чиню бесплатно
